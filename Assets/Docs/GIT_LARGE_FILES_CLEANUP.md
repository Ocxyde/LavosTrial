# GIT LARGE FILES CLEANUP GUIDE
# Purpose: Keep Git repository lean and fast
# Last Updated: 2026-03-09

---

## 🎯 WHY CLEAN LARGE FILES?

Git tracks **every version** of every file. Large files bloat your repository:
- Slower clones and pulls
- Larger `.git` folder
- Wasted bandwidth
- Harder to manage history

**Goal:** Keep Git for **source code only**, use other solutions for large assets.

## 🛠️ HOW TO CLEAN UP

### Step 1: Analyze First

Run the analysis script to see what's tracked:

```powershell
.\analyze-git-large-files.ps1
```

This will show you:
- Files > 1MB currently tracked
- ZIP files in repository
- Files that should be ignored
- Git history issues

### Step 2: Remove from Git Tracking

**Option A: Automated Cleanup**

```powershell
.\cleanup-git-large-files.ps1
```

This safely removes files from Git (keeps local copies).

**Option B: Manual Cleanup**

```powershell
# Remove ZIP files
git rm --cached *.zip

# Remove backup folders
git rm -r --cached Backup/ Backup_Solution/ maze_v0-*/

# Remove build folders
git rm -r --cached Builds/ Builds_*/

# Remove Unity cache
git rm -r --cached Library/ Temp/ Obj/

# Remove scripts (optional)
git rm --cached *.ps1 *.bat
```

### Step 3: Commit Changes

```powershell
git status  # Review what will be removed
git commit -m "chore: remove large/unnecessary files from Git tracking"
```

### Step 4: (Optional) Clean Git History

If large files were in Git history for a long time:

**Using BFG Repo-Cleaner** (Recommended):
```bash
# Download BFG: https://rtyley.github.io/bfg-repo-cleaner/
java -jar bfg.jar --strip-blobs-bigger-than 5M .git
git reflog expire --expire=now --all
git gc --prune=now --aggressive
```

**Using git-filter-repo** (Advanced):
```bash
pip install git-filter-repo
git filter-repo --strip-blobs-bigger-than 5M
```

⚠️ **WARNING:** History rewriting changes commit hashes!
- Only do this on private repositories
- Coordinate with team members if shared

### Step 5: Push to Remote

```powershell
git push
```

If you rewrote history:
```powershell
git push --force
```

## 📦 ALTERNATIVES FOR LARGE FILES

### 1. **Git LFS** (Large File Storage)
For files you need in Git but are large:

```powershell
git lfs install
git lfs track "*.zip"
git lfs track "*.unitypackage"
git lfs track "*.fbx"
```

**Pros:** Integrated with Git, versioned
**Cons:** Requires LFS storage (GitHub/GitLab/Bitbucket)

### 2. **External Storage**
For backups and archives:

- **OneDrive** (built-in on Windows)
- **Google Drive**
- **Dropbox**
- **Synology NAS**
- **External HDD**

**Pros:** Free/cheap, easy sharing
**Cons:** Not versioned (unless using sync clients)

### 3. **Unity Packages**
For Unity assets:

```
Assets → Right-click → Export Package
```

Store `.unitypackage` files externally.

## 📋 PREVENTION STRATEGY

### Update `.gitignore`

Your `.gitignore` already has most patterns. Verify it includes:

```gitignore
# ZIP archives
*.zip

# Backup folders
Backup/
Backup_*/
Backup_Solution/

# Build outputs
Builds/
Builds_*/

# Unity cache
Library/
Temp/
Obj/
UserSettings/

# Scripts (optional)
*.ps1
*.bat
```

### Pre-commit Hook

Create `.git/hooks/pre-commit` (bash script):

```bash
#!/bin/bash

# Block large files (>5MB)
MAX_SIZE=5242880  # 5MB
for file in $(git diff --cached --name-only); do
    size=$(wc -c < "$file")
    if [ $size -gt $MAX_SIZE ]; then
        echo "ERROR: $file is too large ($(($size/1024/1024))MB > 5MB)"
        exit 1
    fi
done

# Block ZIP files
if git diff --cached --name-only | grep -q '\.zip$'; then
    echo "ERROR: ZIP files are not allowed in Git"
    exit 1
fi
```

Make executable:
```bash
chmod +x .git/hooks/pre-commit
```

## 📊 MONITORING

### Check Repository Size

```powershell
# Git folder size
Get-ChildItem .git -Recurse | Measure-Object -Property Length -Sum | 
    Select-Object @{n='Size (MB)';e={[math]::Round($_.Sum/1MB,2)}}

# Largest files in working directory
Get-ChildItem -Recurse | Sort-Object Length -Descending | 
    Select-Object -First 20 Name, @{n='Size (MB)';e={[math]::Round($_.Length/1MB,2)}}
```

### Check Tracked Files Size

```powershell
git ls-files | ForEach-Object {
    [PSCustomObject]@{
        Path = $_
        SizeMB = [math]::Round((Get-Item $_).Length / 1MB, 2)
    }
} | Sort-Object SizeMB -Descending | Select-Object -First 20
```

## 🚨 COMMON ISSUES

### "File still shows in git status after rm --cached"

**Solution:** The file is in a subfolder. Use recursive remove:
```powershell
git rm -r --cached FolderName/
```

### "Git push fails after history rewrite"

**Solution:** Force push (only if you're sure):
```powershell
git push --force
```

### "Teammates can't pull after force push"

**Solution:** They need to re-clone or reset:
```bash
git fetch origin
git reset --hard origin/main
```
