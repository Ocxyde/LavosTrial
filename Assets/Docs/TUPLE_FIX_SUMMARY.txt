🎯 WHY THIS MATTERS:

C# Tuple Rules:
───────────────
1. Unnamed tuples: (int, int)
   - No named properties
   - Can only access via .Item1, .Item2, etc.
   - Cannot use `.x` or `.z`

2. Named tuples: (int x, int z)
   - Have named properties
   - Can access via .x and .z
   - More readable code
   - Better for readability

3. Tuple variables: var tuple = (5, 10)
   - Type is inferred as (int, int) - UNNAMED
   - Must use .Item1, .Item2
   - To make it named: var tuple = (x: 5, z: 10)
